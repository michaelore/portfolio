Toys

I've just begun to sift through the projects I've worked on over the years. I'm starting with my hobbyist projects, those I've done purely for fun. I call them "toys" but they are, in my opinion, just as important as the projects in the other categories. These are the projects that distinguish me the most, as they represent what I'd like to work on in the absence of external pressures such as practicality and working with other developers. That said, I have on multiple occasions adapted code from a "Toy" into a more useful project. I adapted the Spinning Duck into my Robot Simulator and the Erlang/Ajax Toy into some of the code for the ISU Smithsonian Folklife Festival Exhibit. The code for all of these is available at my GitHub account.
